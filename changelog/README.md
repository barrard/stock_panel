# Changelog Notes

This directory is for short working-change notes that sit closer to day-to-day development than `RELEASE_NOTES.md`.

Use this pattern:

- One file per work session or commit batch.
- Filename format: `YYYY-MM-DD.md`.
- Keep entries concise: what changed, why, and any follow-up.
- Treat `RELEASE_NOTES.md` as the higher-level product/history document.

Suggested entry shape:

```md
# 2026-05-02

## Summary
- Short summary of the change set.

## Changes
- Concrete change 1.
- Concrete change 2.

## Notes
- Risks, follow-up, or cleanup items.
```
